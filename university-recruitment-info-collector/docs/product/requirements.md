# 产品需求草稿

## 项目定位

本项目不是单纯的招聘信息爬虫，而是面向高校求职者的招聘信息聚合与岗位匹配工具。

核心价值：

- 自动收集分散在高校人才网、人事处网站、公众号文章中的招聘信息
- 将非结构化招聘公告整理成可检索、可比较的数据
- 用户输入个人背景后，系统自动找出更匹配的岗位
- 通过 LLM 辅助分析岗位要求和用户条件之间的匹配点、不满足项和申请建议

## 核心用户流程

1. 系统定期采集招聘信息
2. 系统清洗、去重并结构化岗位数据
3. 用户进入系统入口，填写个人信息
4. 用户填写求职偏好和限制条件
5. 系统先进行基础规则筛选和排序
6. 系统调用 LLM 对候选岗位进行语义分析
7. 用户查看推荐岗位、匹配理由、风险提示和申请建议

## 用户输入信息

基础背景：

- 最高学历
- 专业
- 研究方向
- 毕业院校
- 毕业时间
- 论文、项目、获奖、工作经历关键词

求职偏好：

- 期望地区
- 期望高校类型
- 期望岗位类型
- 是否接受非编制岗位
- 是否接受博士后、专职科研岗、实验岗等岗位

限制条件：

- 年龄限制
- 编制要求
- 家庭或地域限制
- 配偶工作、子女入学等现实诉求
- 不希望投递的地区或学校类型

## 匹配分析

基础规则匹配：

- 专业或学科方向是否相关
- 学历是否满足
- 地区是否符合偏好
- 截止时间是否仍有效
- 岗位类型是否符合用户偏好
- 明确限制条件是否冲突

LLM 语义分析：

- 分析公告中的隐性要求
- 判断用户研究方向与岗位方向的相关度
- 总结用户适合投递的理由
- 提醒可能不满足的条件
- 给出简历或申请材料侧重点建议

## 输出结果

每个推荐岗位应包含：

- 学校和岗位信息
- 原始公告链接
- 匹配分数
- 匹配理由
- 不确定项或风险点
- 建议补充材料
- LLM 分析摘要

## 阶段规划

第一阶段：

- 高校人才网信息采集
- 招聘公告结构化
- 本地数据导出

第二阶段：

- 用户信息录入入口
- 基础规则匹配
- 推荐结果展示

第三阶段：

- 接入 LLM 分析
- 生成匹配理由和申请建议
- 支持公众号文章来源

第四阶段：

- 增量更新
- 用户订阅和提醒
- 历史投递记录管理
